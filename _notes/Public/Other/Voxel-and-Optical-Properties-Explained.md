# Voxel และสมบัติทางแสง: อธิบายให้เข้าใจง่าย

> เจาะลึกเรื่อง **voxel** และ **สมบัติทางแสง** ที่ทำให้ Glass/Silica Data Storage เก็บข้อมูลได้หลาย bits ต่อ voxel — polarization, retardance, refractive index — อธิบายด้วยภาษาธรรมดา + ภาพประกอบ
>
> 📖 อ่านคู่กับ: [[Glass-Silica-Data-Storage-2026]]

---

## คำศัพท์สำคัญ (Glossary)

ก่อนเริ่ม มาทำความรู้จักคำศัพท์ที่จะเจอบ่อยในบทความนี้กันก่อน:

| คำศัพท์ (English) | แปลไทย | ความหมายแบบเข้าใจง่าย |
|:---|:---|:---|
| **Voxel** | จุดภาพปริมาตร / วอกเซล | Volume Pixel — จุดข้อมูล 3D (เหมือน pixel แต่เป็น 3 มิติ) |
| **Nanograting** | ตะแกรงนาโน / แผ่นระเบียบนาโน | แผ่นบางๆ เรียงกันเป็นระเบียบระดับนาโนเมตร (หนา ~20nm) ที่เลเซอร์สร้างขึ้นในแก้ว |
| **Nanovoid** | รูว่างนาโน | รูเล็กๆ ระดับนาโนที่เกิดจากเลเซอร์ยิงแรงๆ ในแก้ว |
| **Polarization** | โพลาไรเซชัน / การโพลาไรซ์ | ทิศทางที่คลื่นแสงสั่น (แนวตั้ง, แนวนอน, หรือทแยงมุม) |
| **Retardance** | รีทาร์แดนซ์ / ความหน่วงแสง | ผลต่างระหว่างแสงที่ผ่านเร็ว vs ผ่านช้า เมื่อส่องผ่านวัสดุ |
| **Birefringence** | ไบร์เฟรงเจนซ์ / สมบัติหักเหสองเงา | สมบัติที่วัสดุมีดัชนีหักเหต่างกันในแต่ละทิศ → แสงเดินด้วยความเร็วไม่เท่ากัน |
| **Refractive Index (n)** | ดัชนีหักเห | ตัวเลขบอกว่าแสงเดินช้าลงแค่ไหนเมื่อผ่านวัสดุ (อากาศ=1.0, แก้ว≈1.5) |
| **Femtosecond Laser** | เลเซอร์เฟมโตวินาที | เลเซอร์ที่ยิงพัลส์สั้นมาก (10⁻¹⁵ วินาที) ใช้สร้าง voxel ในแก้ว |
| **Fused Silica** | ซิลิกาหลอมเหลว | แก้วบริสุทธิ์ชนิดหนึ่ง (SiO₂) ที่ใช้ใน Project Silica |
| **Borosilicate** | โบโรซิลิเคต | แก้วชนิดหนึ่ง (เช่น Pyrex) ถูกกว่า Fused Silica แต่เก็บข้อมูลได้น้อยกว่า |
| **Isotropic** | ไอโซทรอปิก / เสมอภาคทุกทิศ | สมบัติเท่ากันทุกทิศทาง |
| **Anisotropic** | แอนไอโซทรอปิก / ไม่เสมอภาค | สมบัติต่างกันตามทิศทาง (เช่น nanograting ทำให้แก้วเป็น anisotropic) |
| **Phase Shift** | การเปลี่ยนเฟส | การเลื่อนตำแหน่งของคลื่นแสง (peak ไปชน trough) |
| **Slow Axis** | แกนช้า | ทิศที่แสงผ่านช้ากว่า (เพราะ refractive index สูงกว่า) |
| **Lamellar Nanostructure** | โครงสร้างแผ่นระดับนาโน | โครงสร้างแผ่นบางๆ เรียงซ้อนกัน = nanograting |

## Voxel คืออะไร?

**Voxel = Volume Pixel** — ถ้า pixel คือจุด 2D บนหน้าจอ, voxel คือจุด 3D ในปริมาตร

![Voxel 3D Layers](/assets/img/Other/GlassStorage/voxel-3d-layers.svg)

```
Pixel (2D):          Voxel (3D):
┌───┬───┬───┐       ┌───┐
│ ■ │ □ │ ■ │       │ ■ │  ← ชั้น 1 (z=0)
├───┼───┼───┤       ├───┤
│ □ │ ■ │ □ │       │ □ │  ← ชั้น 2 (z=1)
└───┴───┴───┘       └───┘
 แบน 2 มิติ          ลึก 3 มิติ
```

### Voxel ใน Glass Storage ทำงานยังไง?

**ขั้นตอนทั้งหมด:**

1. **เลเซอร์ยิง** — Femtosecond laser ยิงพัลส์แสง (สั้นมาก ~10⁻¹⁵ วินาที) เข้าไปในแก้ว
2. **เปลี่ยนโครงสร้าง** — พลังงานเลเซอร์ไปเปลี่ยนโครงสร้างระดับนาโนของแก้วตรงจุดนั้น
3. **สร้าง Nanograting หรือ Nanovoid** — แก้วตรงจุดนั้นเกิดการเรียงตัวใหม่:
   - **ถ้าพลังงานปานกลาง** → สร้าง **nanograting** (แผ่นบางๆ เรียงกันเป็นระเบียบ)
   - **ถ้าพลังงานสูงมาก** → สร้าง **nanovoid** (รูว่างเล็กๆ)
4. **การเปลี่ยนแปลงถาวร** — ไม่สลาย ไม่เสื่อม ไม่ต้องใช้ไฟรักษา (อายุ 10,000+ ปี)
5. **อ่านข้อมูล** — ส่องแสงผ่าน voxel → วัดสมบัติทางแสงที่เปลี่ยนไป → แปลงเป็นข้อมูล

> 💡 **คำอธิบายเพิ่ม:** ลองจินตนาการว่าแก้ว = น้ำแข็งใสเต็มก้อน, voxel = จุดที่เราใช้เข็มแหลมๆ เสียบเข้าไปสร้างรอย → แต่ละรอยมีรูปร่าง/ทิศทางต่างกัน → ส่องไฟผ่านแล้วเห็นรอยต่างกัน = ข้อมูลต่างกัน

### ขนาดของ Voxel เทียบกับสิ่งอื่น

| อะไร | ขนาด | เปรียบเทียบ |
|:-----|:-----|:----------|
| เส้นผม | ~70 µm | ใหญ่กว่า voxel ~14-35 เท่า |
| เม็ดทราย | ~60-2000 µm | ใหญ่กว่ามาก |
| เซลล์เม็ดเลือดแดง | ~7 µm | ใหญ่กว่า voxel ~1.5-3.5 เท่า |
| **Voxel (Glass Storage)** | **~2-5 µm** | **ขนาดจริง** |
| แบคทีเรีย | ~1-5 µm | ใกล้เคียง voxel |
| ไวรัส | ~0.02-0.3 µm | เล็กกว่า voxel |
| Nanograting features | **~20 nm** | เล็กกว่า voxel ~100-250 เท่า |

→ Voxel ขนาดใกล้เคียงแบคทีเรีย — ต้องใช้กล้องจุลทรรศน์อ่าน

## "Level" คืออะไร? ทำไมเยอะได้เยอะ bits?

### Level คือค่าที่แตกต่างกันได้ของสมบัติทางแสง

"Level" ไม่ใช่ชั้นของ voxel ในแก้วนะ — แต่หมายถึง **จำนวนค่าที่ต่างกันได้** ของสมบัติทางแสงแต่ละอย่าง

**เปรียบเทียบให้เห็นภาพ:**

```
ธริกน้ำมัน (แยกแค่ 2 level):
  ปิด → OFF → เก็บ "0"
  เปิด → ON  → เก็บ "1"
  → 2 states = 1 bit

ธริกน้ำมันที่หมุนได้ 8 ตำแหน่ง (8 levels):
  ตำแหน่ง 0 → เก็บ "000"
  ตำแหน่ง 1 → เก็บ "001"
  ...
  ตำแหน่ง 7 → เก็บ "111"
  → 8 states = 3 bits
```

### สูตรคำนวณ: bits = log₂(levels)

![Multi-Level Encoding](/assets/img/Other/GlassStorage/multi-level-encoding.svg)

จำนวน bits ที่เก็บได้ = **ลอการิทึมฐาน 2 ของจำนวน levels**

| Levels | คำนวณ | Bits | ตัวอย่างการใช้ |
|:-------|:-------|:-----|:-------------|
| 2 | log₂(2) = 1 | **1 bit** | มี/ไม่มี (เหมือน CD) |
| 4 | log₂(4) = 2 | **2 bits** | 4 ค่าต่างกัน |
| 8 | log₂(8) = 3 | **3 bits** | Project Silica Gen 1 (birefringent) |
| 16 | log₂(16) = 4 | **4 bits** | — |
| 32 | log₂(32) = 5 | **5 bits** | — |
| 64 | log₂(64) = 6 | **6 bits** | — |
| **100** | **log₂(100) ≈ 6.644** | **~6.6 bits** | **Project Silica Gen 2** |
| 256 | log₂(256) = 8 | **8 bits** = 1 byte | — |

### ทำไม 100 levels ถึงได้แค่ 6.6 bits ไม่ใช่ 100 bits?

**เพราะ bits ไม่เท่ากับ levels!** — 1 bit เก็บได้ 2 ค่า (0/1), 2 bits เก็บได้ 4 ค่า (00/01/10/11), n bits เก็บได้ 2ⁿ ค่า

ดังนั้น:
- **100 levels** ต้องการ bits = log₂(100) ≈ **6.644 bits**
- กลับกัน: 6 bits เก็บได้ 2⁶ = 64 levels, 7 bits เก็บได้ 2⁷ = 128 levels
- 100 อยู่ระหว่าง 64 กับ 128 → จึงได้ ~6.6 bits

```
สาธิตให้เห็นชัด:

2 levels:  [0] [1]                           → 1 bit
4 levels:  [00] [01] [10] [11]               → 2 bits  
8 levels:  [000] [001] [010] [011] ... [111]  → 3 bits
16 levels: [0000] [0001] [0010] ... [1111]    → 4 bits
...
100 levels: [??????] ... [??????]              → 6.644 bits
128 levels: [0000000] ... [1111111]            → 7 bits

ถ้ามี 100 level หมายความว่า:
  คุณสามารถแยกค่าออกเป็น 100 ช่วงที่ต่างกันได้
  ซึ่งต้องใช้ "ตัวเลขฐานสอง" 6.644 หลักในการเขียนให้ครบ
```

> 💡 **อะไรทำให้มีได้ 100 levels?** ใน voxel แบบ birefringent, เลเซอร์สามารถควบคุม **มุม polarization** (slow axis orientation) ได้ละเอียดมาก — แทนที่จะแบ่งแค่ 8 มุม (0°, 45°, 90°, ...) ก็แบ่งเป็น 100 มุมที่ต่างกันเล็กๆ (เช่น 0°, 3.6°, 7.2°, ... , 356.4°) → อ่านแยกได้ 100 ค่า → 6.6 bits

### ทำไมไม่เพิ่มเป็น 1,000 levels (≈10 bits)?

**เพราะ noise!** ยิ่งแบ่ง level เยอะ → แต่ละ level ใกล้กันมาก → อ่านสับสน:

```
8 levels:   ช่วงห่าง = 360°/8 = 45°  → อ่านง่าย แยกชัด
100 levels: ช่วงห่าง = 360°/100 = 3.6° → อ่านยากขึ้น แต่ยังไหว
1000 levels: ช่วงห่าง = 360°/1000 = 0.36° → อ่านแยกไม่ออก!

เหมือนไม้บรรทัด:
  ไม้บรรทัด cm → อ่านง่าย แต่ละเส้นห่าง 10mm
  ไม้บรรทัด mm → อ่านยากขึ้น แต่ละเส้นห่าง 1mm  
  ไม้บรรทัด 0.1mm → ตาไม่เห็นแล้ว!
```

**ข้อจำกัดหลัก:**
- **Laser precision** — เลเซอร์ต้องควบคุมมุมได้แม่นยำมาก
- **Detector resolution** — เครื่องอ่านต้องแยกค่าที่ต่างกันน้อยมากได้
- **Material noise** — แก้วมีความไม่สมบูรณ์ตามธรรมชาติ (impurities, stress)
- **Temperature drift** — อุณหภูมิเปลี่ยน → สมบัติแสงเปลี่ยนเล็กน้อย

## ทำไม Voxel เก็บได้หลาย Level?

Voxel ไม่ได้เก็บแค่ "มี/ไม่มี" (เหมือน CD ที่มีแค่ pit/land) แต่เก็บ **สมบัติทางแสง** ได้หลายค่า — แต่ละค่าสามารถแบ่งเป็น multiple levels ได้:

```
CD/DVD (เก็บ 1 bit/จุด):
  pit  = 0      land = 1
  (มี/ไม่มี เท่านั้น)

Glass Storage (เก็บหลาย bit/voxel):
  voxel 1: polarization=45°,  retardance=λ/4  → symbol "5"
  voxel 2: polarization=135°, retardance=λ/2  → symbol "11"
  voxel 3: refractive index change=0.005      → symbol "3"
  ...
  แต่ละสมบัติ = แกนที่เก็บข้อมูลได้ 1 แกน
```

> 💡 **เหมือนสี:** CD = ดินสอดำๆ (มี/ไม่มี), Glass Storage = กล่องสีเครยอน (เลือกสีได้ 100 สี) → สีแต่ละสี = ข้อมูลต่างกัน

## สมบัติทางแสงทั้ง 3 แบบ

### 1. มุม Polarization / Slow Axis Orientation (โพลาไรเซชัน)

![Polarization Explained](/assets/img/Other/GlassStorage/polarization-explained.svg)

**แสงคืออะไร?** แสงเป็นคลื่นแม่เหล็กไฟฟ้า — สั่นได้หลายทิศทาง

```
แสงปกติ (Unpolarized — สั่นทุกทิศ):        Polarized (สั่นทิศเดียว):
    ↗ ↑ ↖                                        ↑ ↑ ↑
  ← ● →                                          ● → →
    ↘ ↓ ↙                                        
                                                 
เหมือนเชือกที่สั่นไปมาทุกทิศ                เหมือนเชือกที่สั่นขึ้น-ลงอย่างเดียว
```

**Polarization (โพลาไรเซชัน) คือ** "ทิศทางที่แสงสั่น" — แสงสามารถสั่นขึ้น-ลง, ซ้าย-ขวา, หรือทแยงมุมก็ได้

**เก็บข้อมูลยังไง:**
- Femtosecond laser สร้าง **nanograting** ในแก้ว — แผ่นบางๆ เรียงกันเป็นระเบียบ
- ทิศทางของ nanograting (มุม θ) ขึ้นกับมุม polarization ของเลเซอร์ที่ยิงเข้าไป
- เวลาอ่าน: ส่องแสงผ่าน → แสงถูกหมุนตามทิศ nanograting → วัดมุมที่ออกมา = รู้ค่าข้อมูล
- สมมติวัดได้ 8 มุม (0°, 45°, 90°, ..., 315°) → **3 bits/voxel**
- วัดได้ 100 มุม → **~6.6 bits/voxel**

```
เลเซอร์ยิงด้วย polarization ต่างกัน:

เลเซอร์ polarized แนวตั้ง ↕     เลเซอร์ polarized แนวนอน ↔
       ↓                                ↓
  ═══╪═══╪═══                    ││││││││││││
  nanograting แนวนอน              nanograting แนวตั้ง
  (θ = 0°) → เก็บ "0"              (θ = 90°) → เก็บ "2"
  
อ่าน: ส่องแสงผ่าน → วัดมุม polarization ที่ออกมา → รู้ค่าข้อมูล
```

> **เปรียบเทียบให้เห็นจริง:** ลองสวมแว่นกันแดด polarized (แว่นตัดแสง) — หมุนแว่นแต่ละมุมจะเห็นแสงต่างกัน → voxel แต่ละอัน "หมุน" แสงต่างมุมกัน → อ่านแยกได้

**ใช้ใน:** Birefringent Voxels → Fused Silica (Project Silica)

---

### 2. Retardance / Optical Path Difference (ความหน่วงแสง)

**Retardance (รีทาร์แดนซ์) คือ** แสงส่องผ่าน voxel → ถูก **หน่วง/ชะลอ** ไม่เท่ากันในแต่ละทิศ

```
แสงเข้า →  [voxel]  → แสงออก
  ↑                    ↑ ↑
  1 ความยาวคลื่น      ทิศ A: ผ่านเร็ว (ไม่หน่วง)
                       ทิศ B: ผ่านช้า (หน่วง λ/4, λ/2, λ...)
                       
  ความต่าง = retardance → วัดได้เป็นตัวเลข
```

**ทำไมถึงหน่วงไม่เท่ากัน?**

Nanograting สร้างโครงสร้างที่มีทิศทาง (**anisotropic** / ไม่เสมอภาค) — เหมือนไม้ไผ่เรียงกัน:
- แสงสั่น **ขนาน** กับแนว nanograting → ผ่านช้า (ถูกหน่วง)
- แสงสั่น **ตั้งฉาก** กับแนว nanograting → ผ่านเร็ว (ไม่ถูกหน่วง)
- ความต่างของ "ช้า vs เร็ว" = **retardance** → วัดได้เป็นตัวเลข

```
Nanograting หนาแน่น (retardance สูง):     Nanograting เบาบาง (retardance ต่ำ):
  ═══╪═══╪═══╪═══╪═══                       ═══╪═══╪═══
  หน่วงมาก → เก็บ "level 7"                 หน่วงน้อย → เก็บ "level 2"
  
ควบคุมด้วย: พลังงานเลเซอร์ → nanograting หนาแน่นมาก/น้อย → retardance สูง/ต่ำ
```

**เก็บข้อมูลยังไง:**
- เลเซอร์ควบคุม **intensity (พลังงานพัลส์)** → ควบคุมความหนาแน่นของ nanograting → ควบคุม retardance
- Retardance แต่ละระดับ = ข้อมูล 1 level
- ยิ่ง resolution ของการวัดสูง → แบ่ง level ได้มาก → เก็บ bits ได้มาก

> **เปรียบเทียบ:** เหมือนกระจกบานเดียวกัน แต่บางจุดหนากว่า → แสงผ่านช้ากว่า → วัดความหนาได้ = อ่านข้อมูลได้

**ใช้ใน:** Birefringent Voxels → Fused Silica (Project Silica) + 5D Memory Crystal (SPhotonix)

> **ความสัมพันธ์กับ Birefringence:** Retardance เกิดจาก **birefringence (ไบร์เฟรงเจนซ์)** — สมบัติของวัสดุที่มี refractive index ต่างกันตามทิศทาง ทำให้แสงเดินด้วยความเร็วต่างกันในแต่ละทิศ → ผลคือ retardance

### 3. Refractive Index Change (การเปลี่ยนดัชนีหักเห)

**Refractive Index (ดัชนีหักเห, n) คือ** ตัวเลขบอกว่า **แสงเดินช้าลงแค่ไหน** เมื่อผ่านวัสดุ

```
อากาศ (n=1.0):     แก้ว (n=1.5):     เพชร (n=2.4):
แสง →  ████████     แสง →  ████████    แสง →  ████████
       เร็ว (300K km/s)  ช้าลง 33%      ช้าลง 58% → "เป็นประกาย"
```

**เก็บข้อมูลยังไง:**

```
แก้วเดิม:  n = 1.500 ทุกจุด (ไม่มีข้อมูล)

หลังเลเซอร์ยิง:
แสง →  [n=1.501] [n=1.505] [n=1.510] [n=1.515]
         level 1    level 2    level 3    level 4
         
เปลี่ยนพลังงานพัลส์ → เปลี่ยน n มาก/น้อย → แต่ละระดับ = ข้อมูล 1 level
```

- Femtosecond laser pulse เปลี่ยน **ความหนาแน่นของแก้ว** ตรงจุด voxel แบบ **isotropic** (เท่ากันทุกทิศ — ไม่มีทิศทาง)
- เปลี่ยนพลังงานพัลส์ → เปลี่ยน refractive index มาก/น้อย → แต่ละระดับ = ข้อมูล 1 level
- อ่านโดยส่องแสงผ่าน + วัด **phase shift** (การเปลี่ยนเฟสของคลื่นแสง)

> **เปรียบเทียบ:** เหมือนแก้วใสที่บางจุด "หนากว่า/บางกว่า" มองไม่เห็นด้วยตาเปล่า แต่ส่องแสงผ่านแล้ววัดการเบี่ยงเบนได้

**ใช้ใน:** Phase Voxels → Borosilicate/Pyrex (Project Silica Gen 2)

> **Phase Voxels vs Birefringent Voxels:** Phase voxels เปลี่ยน refractive index แบบเท่ากันทุกทิศ (isotropic) → เก็บ levels ได้น้อยกว่า แต่ทำงานกับแก้วที่ถูกกว่า (Pyrex) ได้

### ขีดจำกัด: Refractive Index เปลี่ยนได้มากแค่ไหน?

> **คำถาม:** ถ้ายิ่งเปลี่ยน n ได้มาก → แบ่ง level ได้เยอะ → เก็บ bits ได้เยอะ แล้วทำไมไม่ทำให้ n เป็น 5.0 หรือ 1.0?

**คำตอบสั้น:** Femtosecond laser เปลี่ยน refractive index ในแก้วได้ **น้อยมาก** — เปลี่ยนได้แค่ ~0.001 ถึง ~0.05 เท่านั้น ไม่สามารถเปลี่ยนจาก 1.5 เป็น 5.0 ได้

#### ค่า Δn (การเปลี่ยนแปลง refractive index) ที่เป็นไปได้จริง

| พารามิเตอร์ | ค่าที่เป็นไปได้ | หมายเหตุ |
|:-----------|:----------------|:---------|
| Fused Silica ธรรมชาติ (n) | 1.450 | ค่าเริ่มต้น |
| Borosilicate/Pyrex ธรรมชาติ (n) | 1.517 | ค่าเริ่มต้น |
| **Δn จาก femtosecond laser (Phase Voxel)** | **~0.001 - 0.01** | เปลี่ยนนิดเดียว — isotropic |
| **Δn จาก femtosecond laser (Birefringent)** | **~0.01 - 0.05** | มากกว่าเพราะ nanograting สร้าง anisotropy |
| Δn สูงสุดที่เคยทำได้ (Type I) | ~0.005 - 0.02 | พลังงานต่ำ, ไม่ทำลายแก้ว |
| Δn สูงสุดที่เคยทำได้ (Type II) | ~0.01 - 0.05 | พลังงานสูง, สร้าง nanograting |

```
Refractive Index Scale (ค่า n):

0    1.0        1.5                 2.4      5.0
|    |          |                   |        |
|  อากาศ    แก้วธรรมชาติ         เพชร    ???
|            ↓
|     เลเซอร์เปลี่ยนได้แค่นี้
|     ◀━━━━━▶
|     ±0.001~0.05
|     
|     → 1.450 → 1.455 (phase voxel, isotropic)
|     → 1.450 → 1.470 (birefringent, anisotropic)
|     
|     ❌ ทำ 1.450 → 5.0 ไม่ได้!
|     ❌ ทำ 1.450 → 1.0 ไม่ได้!
```

#### ทำไมเปลี่ยนได้น้อยขนาดนี้?

**1. เลเซอร์ไม่ได้ "เปลี่ยนวัสดุ" — เลเซอร์ "ดัดแปลงโครงสร้าง"**

Femtosecond laser ไม่ได้เปลี่ยน SiO₂ เป็นวัสดุอื่น (เช่น เปลี่ยนเป็นเพชร n=2.4 หรือน้ำ n=1.33) — แต่เปลี่ยน **ความหนาแน่น** และ **การเรียงตัว** ของอะตอม SiO₂ เดิม:

```
แก้วเดิม:                          หลังเลเซอร์:
  ○ ○ ○ ○ ○                           ○ ● ○ ○ ○
  ○ ○ ○ ○ ○    เลเซอร์ยิง →          ● ● ○ ○ ○    ← บางจุดหนาแน่นขึ้น
  ○ ○ ○ ○ ○                           ○ ● ○ ○ ○       แต่ยังเป็น SiO₂ เดิม!
  ○ ○ ○ ○ ○                           ○ ○ ○ ○ ○
  
  n = 1.450 เท่ากันทุกจุด            n เปลี่ยนไป ±0.001~0.05
  (amorphous glass)                   (แต่ยังเป็นแก้ว SiO₂)
```

**2. ขีดจำกัดทางกายภาพ**

| ข้อจำกัด | คำอธิบาย |
|:--------|:---------|
| **วัสดุคือ SiO₂** | ไม่ว่าจะยิงเลเซอร์แรงแค่ไหน แก้วก็ยังเป็น SiO₂ → n อยู่ในช่วง 1.4-1.5 เท่านั้น |
| **Phase transition** | ถ้าเลเซอร์แรงเกินไป → แก้ว **แตก/ระเบิด** เป็น nanovoid → ไม่ใช่การเปลี่ยน n แล้ว แต่เป็นการทำลาย |
| **Absorption limit** | พลังงานเลเซอร์ที่สูงเกินไปจะทำให้แก้วดูดกลืนแสง (เปลี่ยนเป็นความร้อน) → ละลาย/แตก |
| **Thermal damage** | อุณหภูมิสูงเกิน → เปลี่ยนโครงสร้างแบบกลับไม่ได้ → เสียข้อมูล |

**3. ทำไมไม่ใช้วัสดุที่ n เปลี่ยนได้มากกว่า?**

| วัสดุ | n ธรรมชาติ | Δn ที่เลเซอร์ทำได้ | ปัญหา |
|:------|:----------|:------------------|:------|
| Fused Silica | 1.450 | 0.001-0.05 | แพง แต่เสถียรสูงสุด |
| Borosilicate (Pyrex) | 1.517 | 0.001-0.02 | ถูก แต่ Δn น้อยกว่า |
| Chalcogenide glass | 2.4-3.0 | 0.01-0.1 | n สูงอยู่แล้ว แต่ **ไม่เสถียร** ต้องใช้ไฟรักษา, เสื่อมเร็ว |
| Lithium Niobate (LiNbO₃) | 2.2-2.3 | 0.001-0.01 | ใช้ใน photonic แต่ **แพงมาก** และเปราะบาง |
| น้ำ | 1.33 | — | ระเหย! ❌ |

> 💡 **สรุป:** ปัญหาไม่ใช่แค่ "ทำ n ให้ห่างจากธรรมชาติมากได้ไหม" แต่เป็น **trade-off** — วัสดุที่ n เปลี่ยนได้มาก (chalcogenide, LiNbO₃) มัก **ไม่เสถียร** หรือ **แพง** → Glass Storage เลือก Fused Silica/Borosilicate เพราะ **เสถียร 10,000+ ปี** แม้ Δn จะน้อย

#### แล้ว Phase Voxel ทำ level ได้กี่ level?

ด้วย Δn ที่เปลี่ยนได้แค่ ~0.01-0.02 ใน Borosilicate:

```
n เริ่มต้น:  1.517
n สูงสุด:    1.537  (เปลี่ยน +0.02)
n ต่ำสุด:    1.507  (เปลี่ยน -0.01)

ช่วงที่ใช้ได้:  1.507 → 1.537 = 0.030

ถ้า detector แยกได้ละเอียด 0.001:
  → 0.030 / 0.001 = 30 levels
  → log₂(30) ≈ 4.9 bits/voxel

ถ้า detector แยกได้แค่ 0.005:
  → 0.030 / 0.005 = 6 levels
  → log₂(6) ≈ 2.6 bits/voxel
```

> **นี่คือเหตุผลที่ Phase Voxels เก็บ bits ได้น้อยกว่า Birefringent Voxels** — ช่วง Δn แคบ + detector มีขีดจำกัด → แบ่ง level ได้น้อย → bits น้อยกว่า

### Nanograting คือ "กลไก" ที่ทำให้ Polarization + Retardance ทำงาน

> **คำถาม:** Nanograting คือการทำ Polarization และ Retardance ใช่ไหม?

**ถูกต้อง!** Nanograting เป็น **โครงสร้างทางกายภาพ** ที่ทำให้เกิดทั้ง polarization encoding และ retardance encoding:

```
Femtosecond Laser ยิงใน Fused Silica
         ↓
   สร้าง Nanograting (โครงสร้างแผ่นนาโน)
         ↓
   ┌─────────────────────────────────────┐
   │  Nanograting ให้ 2 สมบัติพร้อมกัน:    │
   │                                     │
   │  1. ทิศทาง (orientation)             │
   │     → ทำให้เกิด Polarization Change  │
   │     → "มุม" ที่แสงถูกหมุน            │
   │                                     │
   │  2. ความหนาแน่น (density/contrast)  │
   │     → ทำให้เกิด Retardance           │
   │     → "แรง" ที่แสงถูกหน่วง           │
   └─────────────────────────────────────┘
         ↓
   Birefringent Voxel = เก็บได้ 2 มิติ
   (polarization angle + retardance level)
```

**เปรียบเทียบให้เห็นภาพ:**

Nanograting เหมือน **แผ่นฟิลเตอร์** ที่ทั้ง "หมุน" แสง (polarization) และ "หน่วง" แสง (retardance) ไปพร้อมกัน:

```
แสงเข้า →  [Nanograting]  → แสงออก
                │
                ├─ หมุนแสงไปมุม θ     = Polarization angle (มิติ 1)
                └─ หน่วงแสงด้วยปริมาณ δ = Retardance (มิติ 2)
                
ทั้ง 2 อย่างเกิดจาก nanograting ชุดเดียวกัน
แต่ควบคุมแยกกันด้วยพารามิเตอร์เลเซอร์ต่างกัน:
  • มุม θ → ควบคุมด้วย polarization ของเลเซอร์ (ทิศทาง)
  • ปริมาณ δ → ควบคุมด้วย intensity ของเลเซอร์ (พลังงาน)
```

> 💡 **สรุป:** Nanograting ≠ เทคนิคเก็บข้อมูลโดยตรง แต่เป็น **โครงสร้างทางกายภาพ** ที่ทำให้เกิด birefringence → ซึ่งเป็นพื้นฐานให้ทั้ง polarization encoding และ retardance encoding

### Pyrex (Borosilicate) ทำได้แค่ Refractive Index — ทำไม?

> **คำถาม:** ทำไม Pyrex ถึงทำได้แค่ Refractive Index (Phase Voxel) ไม่ได้ทำ Nanograting แบบ Fused Silica?

**เพราะ Nanograting เกิดได้ดีเฉพาะใน Fused Silica!**

```
Fused Silica (SiO₂ บริสุทธิ์):
  ✓ โครงสร้าง amorphous สมบูรณ์ (ไม่มี impurities)
  ✓ Femtosecond laser → ทำให้เกิด self-organized nanograting ได้
  ✓ Nanograting → birefringence → polarization + retardance → 2 มิติ
  ✓ ผล: Birefringent Voxel, เก็บได้ ~6.6 bits/voxel

Borosilicate / Pyrex (SiO₂ + B₂O₃ + Na₂O + Al₂O₃):
  ✗ มีสิ่งเจือปนหลายชนิด (B, Na, Al)
  ✗ Femtosecond laser → nanograting เกิดไม่เป็นระเบียบ/ไม่เกิดเลย
  ✗ แต่! เลเซอร์ยังเปลี่ยนความหนาแน่น (refractive index) ได้ปกติ
  ✓ ผล: Phase Voxel (isotropic Δn), เก็บได้น้อยกว่า แต่ใช้แก้วถูกกว่า
```

**เหตุผลทางเคมี:**

| ปัจจัย | Fused Silica | Borosilicate |
|:------|:------------|:-------------|
| ส่วนประกอบ | SiO₂ บริสุทธิ์ >99.9% | SiO₂ (~80%) + B₂O₃ + Na₂O + Al₂O₃ |
| โครงสร้าง | Amorphous network สม่ำเสมอ | Network มีจุดขาด (network modifiers) |
| Nanograting formation | ✓ Self-organize เป็นระเบียบดีมาก | ✗ สิ่งเจือปนรบกวนการจัดเรียง |
| Δn (isotropic) | ~0.001-0.05 | ~0.001-0.02 |
| Δn (anisotropic/birefringent) | ~0.01-0.1 (ผ่าน nanograting) | แทบไม่เกิด |

> 💡 **เหมือนการสร้างบ้าน:** Fused Silica = ที่ดินเรียบๆ ว่างเปล่า → สร้างอะไรก็ได้ตามแปลน (nanograting เรียบร้อย). Borosilicate = ที่ดินที่มีต้นไม้/หินปนอยู่ → สร้างบ้านเรียบง่ายได้ (phase change) แต่สร้างโครงสร้างซับซ้อนไม่ได้ (nanograting ไม่เป็นระเบียบ)

**Trade-off สรุป:**

```
Fused Silica:
  แพง + เก็บได้เยอะ (4.84 TB/แผ่น)
  ↕
  Nanograting → Polarization + Retardance → ~6.6 bits/voxel
  ↕
  ต้องใช้แก้วบริสุทธิ์พิเศษ

Borosilicate (Pyrex):
  ถูก + เก็บได้น้อยกว่า (2.02 TB/แผ่น)
  ↕
  Phase Change เท่านั้น → Isotropic Δn → ~2-5 bits/voxel
  ↕
  แก้วที่หาซื้อได้ทั่วไป (Pyrex!)
```

---

**ข้อสำคัญ:** Phase Voxels ใน Borosilicate เป็น **นวัตกรรมใหม่ของ Project Silica Gen 2** (Nature 2026) — ก่อนหน้านี้เข้าใจว่าต้องใช้ Fused Silica เท่านั้น แต่ทีม Microsoft พบว่าสามารถใช้ Pyrex + Phase Voxel ได้ แม้ความจุจะต่ำกว่า แต่ต้นทุนต่อ TB ถูกกว่ามาก

## Nanograting คืออะไร? (อธิบายเพิ่มเติม)

![Nanograting Structure](/assets/img/Other/GlassStorage/nanograting-structure.svg)

**Nanograting (ตะแกรงนาโน)** คือโครงสร้างที่เกิดขึ้นในแก้วเมื่อถูก femtosecond laser ยิง — เป็นแผ่นบางๆ (lamellar nanostructures) เรียงกันเป็นระเบียบ:

### ลักษณะทางกายภาพ

| คุณสมบัติ | ค่า |
|:---------|:----|
| ความหนาแต่ละแผ่น | ~20 nm |
| ระยะห่างระหว่างแผ่น (period) | ~100-200 nm |
| จำนวนแผ่นต่อ voxel | ~10-30 แผ่น |
| วัสดุ | Fused Silica (SiO₂) ที่ถูกดัดแปลง |

### ทำไม Nanograting ถึงสำคัญ?

Nanograting เป็น "กุญแจ" ที่ทำให้ voxel เก็บข้อมูลได้หลาย dimension:

1. **ทิศทาง (orientation)** → เก็บ **polarization angle** → ข้อมูล 1 แกน
2. **ความหนาแน่น (density/contrast)** → เก็บ **retardance** → ข้อมูลอีก 1 แกน
3. รวม = 2 แกน × หลาย levels = **bits ต่อ voxel สูง**

```
Nanograting เปรียบเหมือน "ไม้ไผ่เรียงกัน":

ตัดขวาง (ด้านข้าง):         มุมบน (ดูจากด้านบน):
│ ║ │ ║ │ ║ │ ║ │            ═════════════
│ ║ │ ║ │ ║ │ ║ │            ═════════════    แผ่นเรียงแนวนอน
│ ║ │ ║ │ ║ │ ║ │            ═════════════    → θ = 0°
│ ║ │ ║ │ ║ │ ║ │            ═════════════

                             ║║║║║║║║║║║║║║
                             ║║║║║║║║║║║║║║    แผ่นเรียงแนวตั้ง
                             ║║║║║║║║║║║║║║    → θ = 90°
                             ║║║║║║║║║║║║║║

แต่ละทิศทาง = ข้อมูลต่างกัน
```

> 💡 **Fun fact:** Nanograting เกิดจากปรากฏการณ์ **self-organization** — เลเซอร์ไม่ได้ "แกะสลัก" แต่ละแผ่นโดยตรง แต่พลังงานเลเซอร์ทำให้อะตอมในแก้วจัดเรียงตัวเองเป็นระเบียบเอง (self-assembly) — เหมือนคลื่นทะเลที่จัดเรียงทรายเป็นลาย

## สรุป: เปรียบเทียบทั้ง 3 สมบัติ

| สมบัติ | แปลไทย | วัดอะไร | ควบคุมยังไง | ใช้ใน voxel ประเภทไหน | เก็บ bits ได้มากไหม |
|:------|:-------|:--------|:-----------|:---------------------|:-------------------|
| **Polarization angle** | มุมโพลาไรเซชัน | มุมที่แสงถูกหมุน | ทิศทาง polarization ของเลเซอร์ | Birefringent (Fused Silica) | สูง (~6.6 bits) |
| **Retardance** | ความหน่วงแสง | แสงถูกหน่วงแค่ไหน | พลังงานเลเซอร์ → ความหนาแน่น nanograting | Birefringent (Fused Silica) | สูง |
| **Refractive index** | ดัชนีหักเห | แสงเดินช้าลงแค่ไหน | พลังงานเลเซอร์ → ความหนาแน่นแก้วตรงจุด | Phase (Borosilicate) | ปานกลาง |

**Birefringent Voxels** ใช้ polarization + retardance → เก็บได้หลาย bits (สูงสุด ~6.6 bits/voxel) แต่ต้องใช้ Fused Silica แพง

**Phase Voxels** ใช้ refractive index change → เก็บได้น้อยกว่า แต่ใช้กับ Borosilicate (Pyrex) ที่ถูกกว่ามาก → **trade-off ระหว่าง cost vs capacity**

## เหมือนอะไรในชีวิตจริง?

ลองจินตนาการว่า voxel = กระเป๋าเก็บข้อมูล:

```
CD/DVD (1 bit/จุด):
┌──────────┐
│ มี/ไม่มี  │  ← กระเป๋าเล็ก เก็บได้แค่ "ใช่/ไม่ใช่"
└──────────┘

Birefringent Voxel (3-6.6 bits/voxel):
┌──────────┐
│ มุมหมุน  │  ← กระเป๋าใหญ่ เก็บได้หลายมิติ:
│ หน่วงแสง │     มุมหมุน (polarization) และ ความหนา (retardance)
└──────────┘

Phase Voxel (ปานกลาง):
┌──────────┐
│ ความหนา  │  ← กระเป๋ากลาง เก็บได้ 1 มิติ:
│ แก้ว     │     ความหนาแน่น (refractive index) แต่ทำง่ายกว่า
└──────────┘
```

> 💡 **อีกเปรียบเทียบ:** ถ้า CD = เหรียญ (หัว/ก้อย = 2 ค่า = 1 bit) → Voxel 100 levels = ลูกเต้า 100 หน้า (100 ค่า ≈ 6.6 bits) → เดียวกันแต่เก็บได้เยอะกว่า 6 เท่า!
