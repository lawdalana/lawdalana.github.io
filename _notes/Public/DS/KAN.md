# [KAN: Kolmogorov-Arnold Networks สู่การปฏิวัติ Deep Learning](https://arxiv.org/abs/2404.19756)

![KAN01](/assets/img/Other/DS/KAN01.avif)

> **จากยุค "กล่องดำ" สู่ "AI ที่ตีความได้สำหรับวิทยาศาสตร์" ทำไม KAN ถึงเป็นผู้ท้าชิงที่น่ากลัวที่สุดของ MLP?**

---

ในฐานะนักวิจัย AI สิ่งที่เราเผชิญมาตลอดหลายทศวรรษคือการที่สถาปัตยกรรม Deep Learning แทบทั้งหมดล้วนตั้งอยู่บนรากฐานของ Multi-Layer Perceptrons (MLPs) ซึ่งอ้างอิงตามทฤษฎี Universal Approximation Theorem (UAT) แม้ MLPs จะทรงพลังและพาเรามาไกลมาก แต่เราก็ยังติดหล่มอยู่กับปัญหาเรื้อรังสองประการ: **ข้อจำกัดในการตีความ (Interpretability)** และ **ปัญหาคำสาปแห่งมิติ (Curse of Dimensionality)**

แต่แล้วการมาถึงของ **Kolmogorov-Arnold Networks (KANs)** ก็ได้สร้างแรงสั่นสะเทือนครั้งใหญ่ นี่ไม่ใช่แค่การปรับปรุงสถาปัตยกรรมเดิม แต่เป็นการ "รื้อโครงสร้าง" ใหม่ทั้งหมดโดยนำแนวคิดทางคณิตศาสตร์ที่เคยถูกมองข้ามมาใช้

---

## 🔄 1. การเปลี่ยนกระบวนทัศน์: สลับขั้วโหนดและเส้นเชื่อม

![KAN02](/assets/img/Other/DS/KAN02.avif)

เพื่อให้เห็นภาพง่ายๆ ลองจินตนาการว่าใน MLP แบบเดิม เราใช้ "ฟังก์ชันกระตุ้นตายตัว" (เช่น ReLU) วางไว้ที่ **โหนด (Nodes)** และใช้ค่าน้ำหนักคงที่ (Linear Weights) บนเส้นเชื่อม

แต่ KAN พลิกวิธีคิดนี้โดยสิ้นเชิง!

KAN ย้ายการเรียนรู้ไปไว้ที่ **เส้นเชื่อม (Edges)** และเปลี่ยนมันให้กลายเป็น "ฟังก์ชันตัวแปรเดียวที่เรียนรู้ได้" (Univariate Functions) ที่สร้างจาก B-splines ส่วนโหนดมีหน้าที่แค่บวกผลรวมเท่านั้น

การแยกส่วนระหว่าง โครงสร้างเครือข่ายกราฟ (External Degree of Freedom) และ ฟังก์ชันบนเส้นเชื่อม (Internal Degree of Freedom) ช่วยให้ KAN สามารถแกะรอยการทำงานภายในได้อย่างแม่นยำ นี่คือเหตุผลที่ KAN สามารถเอาชนะข้อจำกัดเดิมๆ ได้

## ➰ 2. เจาะลึกกลไก [B-spline: เส้นโค้งแห่งการเรียนรู้](https://www.youtube.com/watch?v=203v61y2o1M)

![KAN03](/assets/img/Other/DS/KAN03.avif)

เพื่อให้เข้าใจความมหัศจรรย์ของ KAN เราต้องมาทำความรู้จักกับ **B-spline (Basis Spline)** ซึ่งเป็นหัวใจสำคัญที่เปลี่ยนเส้นเชื่อมธรรมดาให้กลายเป็น "ฟังก์ชันที่เรียนรู้ได้"

**B-spline คืออะไร?**
B-spline เป็นฟังก์ชันคณิตศาสตร์สำหรับสร้าง "เส้นโค้ง" จากกลุ่ม **จุดควบคุม (Control Points)** จุดเด่นคือ **Local Control (การควบคุมแบบเฉพาะที่)** หมายความว่า หากเราขยับจุดควบคุมใดจุดหนึ่ง มันจะส่งผลกระทบแค่เส้นโค้งบริเวณใกล้เคียงเท่านั้น ไม่ทำให้เส้นโค้งทั้งหมดเพี้ยนไป

**องค์ประกอบหลักของ B-spline**
- **Control Points ($P_i$):** จุดทำหน้าที่เหมือนแม่เหล็กดึงเส้นโค้ง ยิ่งมีจุดมาก ยิ่งปรับ "ความละเอียด" ของเส้นโค้งได้หยุกหยิกมากขึ้น
- **Degree ($p$):** ระดับความสมูทของเส้นโค้ง ยิ่งสูงยิ่งเรียบเนียน (มักล็อกไว้ที่ $p=3$)
- **Knot Vector ($T$):** ชุดตัวเลข "จุดเชื่อม" ที่กำหนดว่าแต่ละ Control Point จะมีอิทธิพลในช่วงไหนบ้าง สมการจำนวนกน็อตคือ $( \text{จำนวนจุดควบคุม} ) + p + 1$

**สมการที่ซ่อนอยู่เบื้องหลัง: Cox-de Boor formula**
สมการนี้ใช้คำนวณ "น้ำหนัก" อิทธิพลของ Control Point แต่ละจุดที่มีต่อตำแหน่งต่างๆ บนเส้นโค้ง ณ ค่าพารามิเตอร์ $t$ (Univariate Parameter):
$$ N_{i,0}(t) = \begin{cases} 1 & \text{if } t_i \le t < t_{i+1} \\ 0 & \text{otherwise} \end{cases} $$
$$ N_{i,p}(t) = \frac{t - t_i}{t_{i+p} - t_i} N_{i,p-1}(t) + \frac{t_{i+p+1} - t}{t_{i+p+1} - t_{i+1}} N_{i+1,p-1}(t) $$

### 🧮 ตัวอย่างการคำนวณเบื้องต้น
เพื่อให้เห็นภาพชัดเจน ลองคำนวณ B-spline แบบ $p=1$ (เส้นตรง) โดยมี 3 Control Points ($P_0, P_1, P_2$) และใช้ Knot Vector $T = [0, 1, 2, 3, 4]$ 
เป้าหมายคือหาตำแหน่งบนเส้นโค้ง ณ เวลา $t = 1.5$

1. **Base Case ($p=0$):** ฟังก์ชันจะให้ค่าเป็น 1 เฉพาะเมื่อ $t$ ตกอยู่ในช่วงนั้น
   เนื่องจาก $1.5$ อยู่ในช่วง $[1, 2)$ ดังนั้นมีเพียง $N_{1,0}(1.5)$ เท่านั้นที่เท่ากับ $1$ (ส่วนจุดอื่นเป็น $0$)
2. **คำนวณน้ำหนัก ($p=1$):** แทนค่า $N_{i,0}$ และ Knot Vector ลงในสมการ Cox-de Boor จะได้:
   - ค่าน้ำหนักของ $P_0 \rightarrow N_{0,1}(1.5) = 0.5$
   - ค่าน้ำหนักของ $P_1 \rightarrow N_{1,1}(1.5) = 0.5$
   - ค่าน้ำหนักของ $P_2 \rightarrow N_{2,1}(1.5) = 0$
3. **สรุปตำแหน่งเส้นโค้ง:** ตำแหน่งที่ $t=1.5$ คือ $C(1.5) = 0.5 P_0 + 0.5 P_1 + 0 P_2$
   *จะเห็นว่าจุด $P_2$ ไม่มีอิทธิพลใดๆ เลย นี่แหละคือคุณสมบัติ Local Control!*

### 🧠 B-spline ในมุมมอง KAN Neural Network (การอัปเดต Backprop)
ในโมเดล KAN ค่าพารามิเตอร์บนเส้นเชื่อมไม่ใช่ค่าน้ำหนัก (Weight) เดี่ยวๆ ดุจเดิม แต่เป็น **Spline Coefficients ($c_i$)** ซึ่งทำหน้าที่ประหนึ่งความสูง (ค่าแกน Y) ของ Control Point
ฟังก์ชันบนเส้นเชื่อมคือ: $f(x) = \sum c_i N_{i,p}(x)$ (ค่า $x$ ทำหน้าที่เหมือนตัวแปร $t$)

**ตัวอย่างการทำงาน Backpropagation ใน KAN (เมื่อ Input $x=1.5$, เป้าหมาย Target $= 4.0$):**
- **Forward Pass:** สมมติโมเดลสุ่มค่าเริ่มต้นพารามิเตอร์มาเป็น $c_0=2.0$, $c_1=4.0$, $c_2=1.0$
  จากตัวอย่างด้านบน น้ำหนักคือ $[0.5, 0.5, 0]$
  โมเดลทายผลลัพธ์ $f(1.5) = (2.0 \times 0.5) + (4.0 \times 0.5) + (1.0 \times 0) = \mathbf{3.0}$
- **Error:** ทาย $3.0$ แต่คำตอบเป้าหมายคือ $4.0$ (ส่วนต่างคือ $1.0$ ต้องดันค่าเพิ่ม)
- **Backward Pass:** โมเดลมองย้อนกลับและพบว่า $c_0$ มีส่วน 50% และ $c_1$ มีส่วน 50% ***แต่ $c_2$ ไม่มีส่วนเลย***
  ดังนั้น Optimizer จะสั่ง **อัปเดตเฉพาะ $c_0$ และ $c_1$ ให้มีค่าเพิ่มขึ้น** (เช่น กลายเป็น $c_0=3.0, c_1=5.0$) และ **ข้าม $c_2$ ไปเลยไม่แตะต้อง**
- **ผลลัพธ์ใหม่:** เมื่อคำนวณรอบใหม่ $f(1.5) = (3.0 \times 0.5) + (5.0 \times 0.5) + 0 = \mathbf{4.0}$ คำตอบเข้าเป้า!

การ "อัปเดตเฉพาะกลุ่มจุด (Local Update)" นี้เอง เป็นสาเหตุที่ทำให้ KAN ประหยัดพลังงานคำนวณในขั้นตอนอัปเดต และไม่กระทบความรู้เก่าที่อยู่ห่างออกไป

## ⚔️ 3. บทพิสูจน์ในสนามจริง: MLP vs KAN

![KAN04](/assets/img/Other/DS/KAN04.avif)

หากเรานำทั้งสองสถาปัตยกรรมมาเทียบกันหมัดต่อหมัด จะเห็นความแตกต่างเชิงกลยุทธ์ที่ชัดเจน:

| คุณลักษณะ | Multi-Layer Perceptron (MLP) | Kolmogorov-Arnold Network (KAN) | ผลกระทบเชิงกลยุทธ์ |
| :--- | :--- | :--- | :--- |
| **รากฐานทฤษฎี** | Universal Approximation Theorem | Kolmogorov-Arnold Theorem | KAN จัดการมิติที่ซับซ้อนภายในโครงสร้างได้ดีกว่า |
| **ตำแหน่ง Activation** | โหนด (Nodes) / ฟังก์ชันตายตัว | ขอบเชื่อม (Edges) / ฟังก์ชันเรียนรู้ได้ | KAN มีความยืดหยุ่นสูง ปรับรูปฟังก์ชันได้ทุกเส้นเชื่อม |
| **พารามิเตอร์** | Linear weight matrices (ค่าคงที่) | B-splines (ฟังก์ชันเส้นโค้งที่เรียนรู้ได้) | เปลี่ยนจากการคูณค่าคงที่เป็นการเรียนรู้ความโค้ง |
| **การทำงานที่โหนด** | Nonlinear activation (ซับซ้อน) | Simple summation (รวมค่าธรรมดา) | ลดความซับซ้อนที่โหนด เพิ่มความโปร่งใส |
| **การขยายสเกล** | เพิ่มความกว้าง/ลึก (ต้องเริ่มฝึกใหม่) | ขยาย Grid ของ Spline (Grid Extension) | เพิ่มความละเอียดได้โดยไม่ต้องนับหนึ่งใหม่ |

> **ผลลัพธ์ที่ซ่อนอยู่:** การใช้ B-splines ทำให้เกิด Local Plasticity หรือการเรียนรู้เฉพาะจุด เมื่ออัปเดตข้อมูลตรงนี้ ข้อมูลส่วนอื่นที่ห่างออกไปจะไม่ลืม (แก้ปัญหา Catastrophic Forgetting)

## 📈 4. กฎการขยายตัว (Scaling Laws) และพลังแห่งความแม่นยำ

![KAN05](/assets/img/Other/DS/KAN05.avif)

ในเชิงประสิทธิภาพการคำนวณ KAN สร้างตัวเลขที่น่าประหลาดใจ:

- **เอาชนะ "คำสาปแห่งมิติ":** ในการแก้ปัญหาสมการ PDE (เช่น สมการ Poisson) โมเดล KAN ขนาดเล็ก (2 ชั้น กว้าง 10) ที่มีพารามิเตอร์ร้อยกว่าตัว ให้ความแม่นยำ (MSE < 0.0000001) เหนือกว่า MLP ที่ใหญ่กว่าถึงร้อยเท่า (4 ชั้น กว้าง 100)
- **กลยุทธ์ Grid Extension ("Staircase Effect"):** นี่คือไม้ตายของ KAN เราสามารถฝึกโครงข่ายหยาบๆ ไปก่อน แล้วเมื่อระบบหาโครงสร้างได้เจอแล้ว จึงค่อยๆ ขยายขนาดความถี่ (Fine-graining) เพื่อเพิ่มความแม่นยำ โดยไม่ต้องเริ่มเทรนใหม่จากศูนย์ ถือเป็นการทำสิ่งที่สถาปัตยกรรมเดิมทำไม่ได้

## 🧠 5. AI Collaborator: ทลายกล่องดำ สู่คณิตศาสตร์ที่มนุษย์เข้าใจ

![KAN06](/assets/img/Other/DS/KAN06.avif)

จุดที่น่าตื่นเต้นที่สุดของ KAN คือความสามารถในการทำงานร่วมกับนักวิทยาศาสตร์ AI ไม่ใช่แค่ผู้ทำนายผลอีกต่อไป แต่เราสามารถทำ **"Human-in-the-loop Surgery"** (การปรับแต่งโครงข่ายด้วยมนุษย์)

- **Symbolification (การแปลงฟังก์ชันเป็นสมการ):** KAN สามารถถูกพล็อตกราฟ และ Snap ฟังก์ชันย่อยให้อยู่ในรูปคณิตศาสตร์มาตรฐาน (เช่น ซายน์, ลอการิทึม, หรือพหุนาม) ได้
- **การค้นพบทางวิทยาศาสตร์:** ในตัวอย่างวิชา ทฤษฎีปม (Knot Theory) KAN ทำนายผลด้วยความแม่นยำถึง 81.6% โดยใช้สัดส่วนพารามิเตอร์เพียงราว 200 ตัว ซึ่งล้มแชมป์หน้าเก่าอย่างโมเดลจาก DeepMind ที่ใช้พารามิเตอร์กว่า 300,000 ตัว (ทำได้ 78%) แถม KAN ยังนำไปสู่การค้นพบสูตรความสัมพันธ์ทางคณิตศาสตร์แบบใหม่
- **Unsupervised Discovery:** KAN ตรวจจับตัวแปรที่มีอิทธิพลต่อกันได้ เพื่อให้นักวิจัยสามารถโฟกัสแต่สิ่งสำคัญ นำไปสู่การค้นพบกฎทฤษฎีใหม่ๆ ในฟิสิกส์ได้อย่างรวดเร็ว

## ⚖️ 6. บทสรุป การตัดสินใจ: งานนี้ใช้ MLP หรือ KAN?

![KAN07](/assets/img/Other/DS/KAN07.avif)

เพื่อให้นำไปใช้งานได้จริง นี่คือ Decision Checklist สั้นๆ ว่าเมื่อไหร่ควรใช้อะไรคะ:

1. **ต้องการความแม่นยำสูงในมิติต่ำ (Low-dimensional)?** 👉 เลือก **KAN**
2. **ต้องการอธิบายผลลัพธ์ (Interpretability) และเข้าใจกลไกภายใน?** 👉 เลือก **KAN**
3. **ต้องการโมเดลที่มีขนาดเล็กสำหรับอุปกรณ์ขนาดจำกัด (Memory constraint)?** 👉 เลือก **KAN**
4. **ทำ Continual Learning (เรียนรู้ต่อเนื่องโดยไม่ลืมความรู้เก่า)?** 👉 เลือก **KAN**
5. **ต้องการความเร็วในการเทรนที่สุด ข้อมูลมหาศาล สเกลระดับโครงข่ายหลัก?** 👉 เลือก **MLP** (ปัจจุบัน KAN ยังใช้เวลาเทรนช้ากว่าถึง 10 เท่า)

🔥 **บทวิเคราะห์ทิ้งท้าย:**
สถาปัตยกรรม KAN เปรียบเสมือนปฐมบทของ *"Language Model for Science"* ที่ใช้ภาษาสมการคณิตศาสตร์มาเป็นตัวกลางสื่อสารระหว่างแพทเทิร์นของข้อมูลกับมนุษย์ แม้จะยังทำงานช้ากว่าในเชิงวิศวกรรมการรัน Batch บน GPU แต่ศักยภาพในการเป็นสะพานทอดผ่านความซับซ้อนไปสู่วิธีคิดโปร่งใสสไตล์ "Glass Box" ทำให้ KAN ได้กลายเป็นเข็มทิศเล่มใหม่ที่โลก AI ไม่มีวันหันหลังกลับไปมองแบบเดิมอีกแน่นอน
