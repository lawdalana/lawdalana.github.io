# KAN: Kolmogorov-Arnold Networks สู่การปฏิวัติ Deep Learning

> **จากยุค "กล่องดำ" สู่ "AI ที่ตีความได้สำหรับวิทยาศาสตร์" ทำไม KAN ถึงเป็นผู้ท้าชิงที่น่ากลัวที่สุดของ MLP?**

---

ในฐานะนักวิจัย AI สิ่งที่เราเผชิญมาตลอดหลายทศวรรษคือการที่สถาปัตยกรรม Deep Learning แทบทั้งหมดล้วนตั้งอยู่บนรากฐานของ Multi-Layer Perceptrons (MLPs) ซึ่งอ้างอิงตามทฤษฎี Universal Approximation Theorem (UAT) แม้ MLPs จะทรงพลังและพาเรามาไกลมาก แต่เราก็ยังติดหล่มอยู่กับปัญหาเรื้อรังสองประการ: **ข้อจำกัดในการตีความ (Interpretability)** และ **ปัญหาคำสาปแห่งมิติ (Curse of Dimensionality)**

แต่แล้วการมาถึงของ **Kolmogorov-Arnold Networks (KANs)** ก็ได้สร้างแรงสั่นสะเทือนครั้งใหญ่ นี่ไม่ใช่แค่การปรับปรุงสถาปัตยกรรมเดิม แต่เป็นการ "รื้อโครงสร้าง" ใหม่ทั้งหมดโดยนำแนวคิดทางคณิตศาสตร์ที่เคยถูกมองข้ามมาใช้

---

## 🔄 1. การเปลี่ยนกระบวนทัศน์: สลับขั้วโหนดและเส้นเชื่อม

เพื่อให้เห็นภาพง่ายๆ ลองจินตนาการว่าใน MLP แบบเดิม เราใช้ "ฟังก์ชันกระตุ้นตายตัว" (เช่น ReLU) วางไว้ที่ **โหนด (Nodes)** และใช้ค่าน้ำหนักคงที่ (Linear Weights) บนเส้นเชื่อม

แต่ KAN พลิกวิธีคิดนี้โดยสิ้นเชิง!

KAN ย้ายการเรียนรู้ไปไว้ที่ **เส้นเชื่อม (Edges)** และเปลี่ยนมันให้กลายเป็น "ฟังก์ชันตัวแปรเดียวที่เรียนรู้ได้" (Univariate Functions) ที่สร้างจาก B-splines ส่วนโหนดมีหน้าที่แค่บวกผลรวมเท่านั้น

การแยกส่วนระหว่าง โครงสร้างเครือข่ายกราฟ (External Degree of Freedom) และ ฟังก์ชันบนเส้นเชื่อม (Internal Degree of Freedom) ช่วยให้ KAN สามารถแกะรอยการทำงานภายในได้อย่างแม่นยำ นี่คือเหตุผลที่ KAN สามารถเอาชนะข้อจำกัดเดิมๆ ได้

---

## ⚔️ 2. บทพิสูจน์ในสนามจริง: MLP vs KAN

หากเรานำทั้งสองสถาปัตยกรรมมาเทียบกันหมัดต่อหมัด จะเห็นความแตกต่างเชิงกลยุทธ์ที่ชัดเจน:

| คุณลักษณะ | Multi-Layer Perceptron (MLP) | Kolmogorov-Arnold Network (KAN) | ผลกระทบเชิงกลยุทธ์ |
| :--- | :--- | :--- | :--- |
| **รากฐานทฤษฎี** | Universal Approximation Theorem | Kolmogorov-Arnold Theorem | KAN จัดการมิติที่ซับซ้อนภายในโครงสร้างได้ดีกว่า |
| **ตำแหน่ง Activation** | โหนด (Nodes) / ฟังก์ชันตายตัว | ขอบเชื่อม (Edges) / ฟังก์ชันเรียนรู้ได้ | KAN มีความยืดหยุ่นสูง ปรับรูปฟังก์ชันได้ทุกเส้นเชื่อม |
| **พารามิเตอร์** | Linear weight matrices (ค่าคงที่) | B-splines (ฟังก์ชันเส้นโค้งที่เรียนรู้ได้) | เปลี่ยนจากการคูณค่าคงที่เป็นการเรียนรู้ความโค้ง |
| **การทำงานที่โหนด** | Nonlinear activation (ซับซ้อน) | Simple summation (รวมค่าธรรมดา) | ลดความซับซ้อนที่โหนด เพิ่มความโปร่งใส |
| **การขยายสเกล** | เพิ่มความกว้าง/ลึก (ต้องเริ่มฝึกใหม่) | ขยาย Grid ของ Spline (Grid Extension) | เพิ่มความละเอียดได้โดยไม่ต้องนับหนึ่งใหม่ |

> **ผลลัพธ์ที่ซ่อนอยู่:** การใช้ B-splines ทำให้เกิด Local Plasticity หรือการเรียนรู้เฉพาะจุด เมื่ออัปเดตข้อมูลตรงนี้ ข้อมูลส่วนอื่นที่ห่างออกไปจะไม่ลืม (แก้ปัญหา Catastrophic Forgetting)

## 📈 3. กฎการขยายตัว (Scaling Laws) และพลังแห่งความแม่นยำ

ในเชิงประสิทธิภาพการคำนวณ KAN สร้างตัวเลขที่น่าประหลาดใจ:

- **เอาชนะ "คำสาปแห่งมิติ":** ในการแก้ปัญหาสมการ PDE (เช่น สมการ Poisson) โมเดล KAN ขนาดเล็ก (2 ชั้น กว้าง 10) ที่มีพารามิเตอร์ร้อยกว่าตัว ให้ความแม่นยำ (MSE < $10^{-7}$) เหนือกว่า MLP ที่ใหญ่กว่าถึงร้อยเท่า (4 ชั้น กว้าง 100)
- **กลยุทธ์ Grid Extension ("Staircase Effect"):** นี่คือไม้ตายของ KAN เราสามารถฝึกโครงข่ายหยาบๆ ไปก่อน แล้วเมื่อระบบหาโครงสร้างได้เจอแล้ว จึงค่อยๆ ขยายขนาดความถี่ (Fine-graining) เพื่อเพิ่มความแม่นยำ โดยไม่ต้องเริ่มเทรนใหม่จากศูนย์ ถือเป็นการทำสิ่งที่สถาปัตยกรรมเดิมทำไม่ได้

---

## 🧠 4. AI Collaborator: ทลายกล่องดำ สู่คณิตศาสตร์ที่มนุษย์เข้าใจ

จุดที่น่าตื่นเต้นที่สุดของ KAN คือความสามารถในการทำงานร่วมกับนักวิทยาศาสตร์ AI ไม่ใช่แค่ผู้ทำนายผลอีกต่อไป แต่เราสามารถทำ **"Human-in-the-loop Surgery"** (การปรับแต่งโครงข่ายด้วยมนุษย์)

- **Symbolification (การแปลงฟังก์ชันเป็นสมการ):** KAN สามารถถูกพล็อตกราฟ และ Snap ฟังก์ชันย่อยให้อยู่ในรูปคณิตศาสตร์มาตรฐาน (เช่น ซายน์, ลอการิทึม, หรือพหุนาม) ได้
- **การค้นพบทางวิทยาศาสตร์:** ในตัวอย่างวิชา ทฤษฎีปม (Knot Theory) KAN ทำนายผลด้วยความแม่นยำถึง 81.6% โดยใช้สัดส่วนพารามิเตอร์เพียง $\approx 200$ ตัว ซึ่งล้มแชมป์หน้าเก่าอย่างโมเดลจาก DeepMind ที่ใช้พารามิเตอร์กว่า $3\times10^5$ ตัว (ทำได้ 78%) แถม KAN ยังนำไปสู่การค้นพบสูตรความสัมพันธ์ทางคณิตศาสตร์แบบใหม่
- **Unsupervised Discovery:** KAN ตรวจจับตัวแปรที่มีอิทธิพลต่อกันได้ เพื่อให้นักวิจัยสามารถโฟกัสแต่สิ่งสำคัญ นำไปสู่การค้นพบกฎทฤษฎีใหม่ๆ ในฟิสิกส์ได้อย่างรวดเร็ว

---

## ⚖️ 5. บทสรุป การตัดสินใจ: งานนี้ใช้ MLP หรือ KAN?

เพื่อให้นำไปใช้งานได้จริง นี่คือ Decision Checklist สั้นๆ ว่าเมื่อไหร่ควรใช้อะไรคะ:

1. **ต้องการความแม่นยำสูงในมิติต่ำ (Low-dimensional)?** 👉 เลือก **KAN**
2. **ต้องการอธิบายผลลัพธ์ (Interpretability) และเข้าใจกลไกภายใน?** 👉 เลือก **KAN**
3. **ต้องการโมเดลที่มีขนาดเล็กสำหรับอุปกรณ์ขนาดจำกัด (Memory constraint)?** 👉 เลือก **KAN**
4. **ทำ Continual Learning (เรียนรู้ต่อเนื่องโดยไม่ลืมความรู้เก่า)?** 👉 เลือก **KAN**
5. **ต้องการความเร็วในการเทรนที่สุด ข้อมูลมหาศาล สเกลระดับโครงข่ายหลัก?** 👉 เลือก **MLP** (ปัจจุบัน KAN ยังใช้เวลาเทรนช้ากว่าถึง 10 เท่า)

🔥 **บทวิเคราะห์ทิ้งท้าย:**
สถาปัตยกรรม KAN เปรียบเสมือนปฐมบทของ *"Language Model for Science"* ที่ใช้ภาษาสมการคณิตศาสตร์มาเป็นตัวกลางสื่อสารระหว่างแพทเทิร์นของข้อมูลกับมนุษย์ แม้จะยังทำงานช้ากว่าในเชิงวิศวกรรมการรัน Batch บน GPU แต่ศักยภาพในการเป็นสะพานทอดผ่านความซับซ้อนไปสู่วิธีคิดโปร่งใสสไตล์ "Glass Box" ทำให้ KAN ได้กลายเป็นเข็มทิศเล่มใหม่ที่โลก AI ไม่มีวันหันหลังกลับไปมองแบบเดิมอีกแน่นอน
